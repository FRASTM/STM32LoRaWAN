\begin{Indent}\textbf{ OTAA Join}\par
{\em \label{classSTM32LoRaWAN_joinOTAA}%
\Hypertarget{classSTM32LoRaWAN_joinOTAA}%


This method can be used to join the network using the OTAA (over-\/the-\/air-\/activation) method, where the device uses the credentials specified here to negotiate a session (including session address and encryption keys) with the network by exchanging messages.

The app\+EUI and app\+Key are mandatory, the dev\+EUI can be omitted, in which case the dev\+EUI baked into the chip is used.

For both EUIs and the key, you can pass a hex-\/encoded (MSB-\/first) string ({\ttfamily String} object or {\ttfamily const char$\ast$}), for the EUIs also a raw integer (64-\/bits).

These methods will perform multiple join attempts and block until the join completes successfully, or a timeout (60 seconds) occurs.

There are some subtle differences with MKRWAN in how the join process works\+:
\begin{DoxyItemize}
\item MKRWAN returns directly after the timeout, while this library finishes the running join attempt before join\+OTAA returns.
\item With MKRWAN the module continues doing join attempts in the background, while this library stops attempting them after join\+OTAA returns.
\item With MKRWAN the module automatically decreases/alternates the datarate in a region-\/specific way for all regions, this library uses the configured datarate (with {\ttfamily datarate()}) for most regions, but uses fixed datarates (according to Lo\+Ra\+WAN regional parameters) for US915/\+AU915/\+AS923.
\end{DoxyItemize}

\begin{Indent}\textbf{ ABP Join}\par
{\em \label{classSTM32LoRaWAN_joinABP}%
\Hypertarget{classSTM32LoRaWAN_joinABP}%


This method can be used to join the network using the ABP method, where the session address and keys are preconfigured and no exchange with the network is needed before data can be sent.

For the address and both keys, you can pass a hex-\/encoded (MSB-\/first) string ({\ttfamily String} object or {\ttfamily const char$\ast$}), for the address also a raw integer (32-\/bits).

\begin{DoxyWarning}{Warning}
This library does {\itshape not} preserve frame counters in non-\/volatile storage, so it starts at zero after every reset. This only works when the server disables framecounter-\/based replay attacks, otherwise only the first session will work and data will be dropped after the first reset.
\end{DoxyWarning}
\begin{DoxyNote}{Note}
An ABP join returns immediately, it does not need to wait for the network, so there is no separate non-\/blocking/async version of this method. 
\end{DoxyNote}

\begin{Indent}\textbf{ Packet sending}\par
{\em \label{classSTM32LoRaWAN_sending}%
\Hypertarget{classSTM32LoRaWAN_sending}%


These methods allow sending a data packet.

After a join was completed, sending a packet consists of calling \mbox{\hyperlink{classSTM32LoRaWAN_a53de6fa3a4971cde2ff3da8331b877dc}{begin\+Packet()}}, writing data to using the Stream write methods, and calling \mbox{\hyperlink{classSTM32LoRaWAN_a816dd9c8b223bda9380971c5a1dab377}{end\+Packet()}} to finish up and send the packet.

The port number to use for the packet can be set using \mbox{\hyperlink{classSTM32LoRaWAN_af342c59031d83d102bc58672f844f64f}{set\+Port()}} (to be called before \mbox{\hyperlink{classSTM32LoRaWAN_a816dd9c8b223bda9380971c5a1dab377}{end\+Packet()}}). \begin{Indent}\textbf{ Stream reading}\par
{\em \label{classSTM32LoRaWAN_stream}%
\Hypertarget{classSTM32LoRaWAN_stream}%


These are standard methods defined by the Arduino Stream class to allow reading received data.

In addition to the methods listed here, all methods offered by the Arduino Stream class are also available, see \href{https://www.arduino.cc/reference/en/language/functions/communication/stream/}{\texttt{ https\+://www.\+arduino.\+cc/reference/en/language/functions/communication/stream/}}

After a packet is received, the \mbox{\hyperlink{classSTM32LoRaWAN_a9c28e7976a749ba1871f5df806b21882}{available()}} method can be used to query how much bytes are in the packet (or after some bytes were read, how many are left to be read), and various \mbox{\hyperlink{classSTM32LoRaWAN_aa04eea06439708bf70cfa524325f427f}{read()}} versions can be used to read the data.

\begin{DoxyNote}{Note}
If data is left unread when a new packet is received, the new data will be appended to the unread data and it becomes impossible to query where the first packet ends and the second begins. It is recommended to always fully read any received data before transmitting a new packet (which is, in class A Lo\+Ra\+WAN, the only time a new packet can be received). 
\end{DoxyNote}

{\footnotesize\ttfamily bool STM32\+Lo\+Ra\+WAN\+::set\+ADR (\begin{DoxyParamCaption}\item[{bool}]{adr }\end{DoxyParamCaption})}

Enable or disable ADR (automatic datarate).

This is enabled by default and allows the network to increase the datarate used (when sending downlink commands) when transmissions from this device are received with sufficient quality that a higher datarate is expected to still be received. In addition, it causes the device to request periodic confirmations and lets it reduce the datarate when transmissions are left unconfirmed.

Together, these two mechanisms should ensure the highest possible datarate is used, maximizing bandwidth and/or reducing spectrum usage (time on air). \mbox{\Hypertarget{classSTM32LoRaWAN_a6055d4c91f0c5f3b0671142c5b3cc3aa}\label{classSTM32LoRaWAN_a6055d4c91f0c5f3b0671142c5b3cc3aa}}

Registers a callback that is called whenever there is some work to do and the \mbox{\hyperlink{classSTM32LoRaWAN_a2c207de2ede561d57ef26af61991cffa}{maintain()}} function must be called soon.

\begin{DoxyWarning}{Warning}
This callback is called from an interrupt handler, so it should not do any work and definitely {\itshape not} call {\ttfamily \mbox{\hyperlink{classSTM32LoRaWAN_a2c207de2ede561d57ef26af61991cffa}{maintain()}}}, but just set a flag, enable a task, or something similarly short, and make sure to do that in a interrupt-\/safe manner.
\end{DoxyWarning}
When using this callback, it is not needed to call the {\ttfamily \mbox{\hyperlink{classSTM32LoRaWAN_a2c207de2ede561d57ef26af61991cffa}{maintain()}}} method unless this callback is called, allowing applications to be more efficient, or use sleeping (but care should be taken to prevent race conditions).

Only one callback can be active at the same time, so any previously configured callback is replaced by the new one passed.
